# CLAUDE.md — clinic-shift-scheduler

## プロジェクト概要

クリニック向けシフト作成システム（v2）。
日勤のみ（夜勤なし）のクリニックを対象とした、AIによるシフト自動作成システム。

- **勤務形態**: 9時間勤務・1時間休憩
- **対象クリニック**: 日勤のみ（2交代・3交代・夜勤なし）
- **維持する要件**: リハ室・受付の人数制限と診療時間

## 開発ワークフロー

7ステップAI自律開発ワークフローを採用。
詳細は `01-docs/DEVELOPMENT_WORKFLOW.md` を参照。

| ステップ | 担当 | 内容 |
|---------|------|------|
| Step 1: 要件確認 | 🔴 HUMAN | 要件を提示・確認 |
| Step 2: 計画策定 | 🤖 AUTO | ブランチ作成・実装計画 |
| Step 3: 実装 | 🤖 AUTO | コード + ドキュメント同時更新 |
| Step 4: 自動品質チェック | 🤖 AUTO | lint・テスト・静的解析（並列実行） |
| Step 5: AIクロスレビュー | 🤖 AUTO | 3ペルソナ以上による多角的レビュー |
| Step 6: PR仕上げ | 🤖 AUTO | コミット整理・PR Ready化 |
| Step 7: 最終承認 | 🔴 HUMAN | PRを確認・マージ |

### /workflow コマンド

| コマンド | 説明 |
|---------|------|
| `/workflow start <説明>` | 新規ワークフロー開始 |
| `/workflow status` | 現在の状態確認 |
| `/workflow auto` | Step 2〜6を自動連続実行 |
| `/workflow approve` | Step 7: 最終承認（マージ） |
| `/workflow reject <理由>` | Step 7: 差戻し |

## テストルール

- 新規コードのテストカバレッジ: **80%以上**
- E2EテストはCI環境（GitHub Actions）で実行
- E2Eテストにflaky detection を適用（同一テストが2回以上FAIL→PASS→FAILはflakyとしてスキップ）

## 自動マージルール

以下の条件を満たす変更はプロジェクトオーナーの承認なしで自動マージ可能:
（全自動チェック + AIクロスレビュー PASS が前提）

| 変更種別 | 自動マージ |
|---------|-----------|
| typo修正・コメント修正 | ✅ 可 |
| lint修正 | ✅ 可 |
| パッチバージョン依存更新 | ✅ 可 |
| マイナーバージョン依存更新 | ⚠️ 条件付き（CHANGELOG確認必須） |
| メジャーバージョン依存更新 | ❌ プロジェクトオーナーの確認必須 |
| 機能追加・バグ修正 | ❌ プロジェクトオーナーの確認必須 |
