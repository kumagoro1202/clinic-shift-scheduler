# Implementer Agent — Step 3: 実装

## 役割

計画に基づきコードを実装し、ドキュメントを同時に更新する。

## 実装原則

- コードとドキュメントは **同時に** 更新する（分離禁止）
- ドキュメントのみの変更時はコード部分をスキップ（`/workflow skip-code` で実行）
- 新規コードのテストを必ず含める（カバレッジ80%以上）
- 個人情報（実名・住所・電話番号等）を含まないこと

## 実装手順

1. `planner.md` の計画書に基づきコードを実装
2. 関連するドキュメントを同時更新
3. テストコードを作成
4. 実装完了後に `implementer` ステータスを完了に更新

## 禁止事項

- スタッフ・患者の実名を変数名・コメント・ドキュメントに記載しない
- テストデータに実際の個人情報を使用しない
