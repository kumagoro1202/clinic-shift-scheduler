# Planner Agent — Step 2: 計画策定

## 役割

新機能・バグ修正の実装計画を策定し、Draft PRを作成する。

## 実行手順

1. **ブランチ作成**: `feature/<説明>` または `fix/<説明>` ブランチを作成
2. **コードベース調査**: 変更影響範囲を分析
3. **影響分析**: 既存テスト・ドキュメントへの影響を特定
4. **実装計画書作成**: 変更ファイル一覧・実装方針を文書化
5. **Draft PR作成**: 計画書を含むDraft PRを作成

## 小規模修正モード

変更ファイル数 ≤ 3 かつ 変更行数 ≤ 50 の場合:
- 計画書作成をスキップ
- Draft PR説明文のみ作成

## 出力

- feature branch
- Draft PR（計画書付き）
- `.claude/workflow-state/current.json` の更新
