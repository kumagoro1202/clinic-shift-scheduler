# Quality Checker Agent — Step 4: 自動品質チェック

## 役割

lint・テスト・静的解析を並列実行し、品質を自動で担保する。

## チェック項目（並列実行）

| チェック | ツール（言語決定後に更新） | 失敗時の対応 |
|---------|--------------------------|-------------|
| コードlint | ruff / eslint / spotless | 自動修正 → 再実行 |
| ドキュメントlint | markdownlint | 自動修正 → 再実行 |
| ユニットテスト | pytest / npm test / mvn test | エラー解析 → 修正 → 再実行 |
| E2Eテスト | Playwright | Flaky検出 → 再実行（最大2回） |
| 静的解析 | pyright / tsc / checkstyle | 型エラー修正 → 再実行 |
| 個人情報チェック | scripts/pii-check.sh | 匿名化 → 再実行 |

## 自動修正ループ

```text
チェック実行
  ├── 全PASS → Step 5へ
  └── FAIL → 修正サイクル（最大3回）
       ├── エラー内容を解析
       ├── 修正方針を決定
       ├── 自動修正を適用
       └── チェック再実行
            ├── 3回以内 → 再チェック
            └── 3回超過 → Step 7へエスカレーション
```

## Flakyテスト検出

同一テストが2回以上 FAIL→PASS→FAIL を繰り返す場合:
- flakyとしてスキップ
- レポートに記録（修正の必要あり）
- 他のテストが全PASSなら次のステップに進む

## テストカバレッジ

新規コード: 80%以上（未達はMinor警告として記録）
