# ADR-004: 7ステップAI自律開発ワークフローの採用

| 項目 | 内容 |
|------|------|
| **ステータス** | 承認済み |
| **日付** | 2026-03-01 |
| **意思決定者** | プロジェクトオーナー（方針指示）、設計レビューエージェント（設計）、3ペルソナ（WP1/WP2/WP3）（レビュー） |

## コンテキスト

v1の21ステップワークフロー（22エージェント定義）は検討段階で実運用されなかった。
プロジェクトオーナーより「人間の確認を最小限にしつつ確実に品質担保」する新ワークフローの設計を指示。

## 決定事項

7ステップ構成のAI自律開発ワークフローを新規設計:

```text
Step 1: 要件確認（🔴 HUMAN）
Step 2: 計画策定（🤖 AUTO）
Step 3: 実装（🤖 AUTO）
Step 4: 自動品質チェック（🤖 AUTO）
Step 5: AIクロスレビュー（🤖 AUTO）
Step 6: PR仕上げ（🤖 AUTO）
Step 7: 最終承認（🔴 HUMAN）
```

付帯決定:

- AI自動修正ループ（Step 4で最大3回リトライ）
- 品質チェック並列実行（lint & test & static analysis）
- 小規模修正モード（Step 2省略、ファイル3以下 × 50行以下）
- 段階的自動マージルール（typo/lint修正は人間承認不要）

## 代替案と却下理由

| 代替案 | 却下理由 |
|--------|---------|
| v1の21ステップをそのまま移植 | 実運用実績がない。22エージェントの保守コスト大 |
| v1を5-10ステップに部分統合 | 中途半端。3人間ゲートの構造的問題は残る |
| 人間レビューなし（完全自動） | 要件誤解のリスクがある。最低限の人間確認は必要 |

## 結果と影響

- 人間の待ち時間: 最低3回 → 最低2回（要件確認 + 最終承認）
- エージェントファイル: 22 → 5
- 自動化率: 約70% → 約86%

## 根拠

1. v1の21ステップは検討段階の産物で実運用実績がない（プロジェクトオーナー確認済み）
2. `/workflow auto` で要件確認後〜最終承認前まで自律実行可能
3. AI自動修正ループにより、人間の待ち時間を劇的に削減
4. AIクロスレビュー（3ペルソナ以上）で品質担保。人間は最終サマリのみ確認

## 自動マージルール（段階的）

| 変更種別 | 自動マージ | 条件 |
|---------|-----------|------|
| typo修正・コメント修正 | ✅ 可 | 全自動チェック + AIレビュー PASS |
| lint修正 | ✅ 可 | 全自動チェック + AIレビュー PASS |
| パッチバージョン依存更新 | ✅ 可 | 全自動チェック + AIレビュー PASS |
| マイナーバージョン依存更新 | ⚠️ 条件付き | 全自動チェック + AIレビュー PASS + CHANGELOG確認 |
| メジャーバージョン依存更新 | ❌ 不可 | 人間確認必須 |
| 機能追加・バグ修正 | ❌ 不可 | 人間確認必須 |
