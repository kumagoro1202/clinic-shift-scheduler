# ADR-002: テストディレクトリの構成

| 項目 | 内容 |
|------|------|
| **ステータス** | 承認済み |
| **日付** | 2026-03-01 |
| **意思決定者** | 設計レビューエージェント（提案）、4ペルソナ（レビューで合意） |

## コンテキスト

v1では `03-tests/backend/`, `03-tests/e2e/`, `03-tests/frontend/` とアプリケーション層別に分類していた。
v2では実装言語が未定のため、言語非依存の分類方法が必要。

## 決定事項

テスト種別による分類を採用: `03-tests/unit/`, `03-tests/integration/`, `03-tests/e2e/`

## 代替案と却下理由

| 代替案 | 却下理由 |
|--------|---------|
| v1のアプリケーション層別分類（backend/e2e/frontend） | 実装言語未定の段階で層別分類をするとリファクタリングが必要になる |
| 単一のtestsディレクトリ | テスト種別が不明確になり、CIジョブとの対応が困難 |

## 結果と影響

- テスト種別分類は実装言語に依存しない
- CIジョブとの対応が直感的（unit → 高速実行、e2e → PRのみ等）
- テストピラミッドの構造が物理的に可視化される

## 根拠

1. unit/integration/e2e はテスト種別の標準的な分類
2. CIでの実行戦略（並列・シリアル・トリガー条件）がディレクトリ名から自明
3. 実装言語変更時のディレクトリ変更が不要
