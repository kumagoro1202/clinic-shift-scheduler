# 開発ワークフロー定義書

## 概要

7ステップAI自律開発ワークフロー。
人間の確認を最小限にしつつ、AIによる自律的な品質担保を実現する。

## 全体フロー

```text
┌─────────────────────────────────────────────────────────────┐
│ Step 1: 要件確認 (🔴 HUMAN GATE)                           │
│   人間が要件を提示。AIが要件を構造化し、確認を取る。          │
│   Output: 承認済み要件（タスク記述 or REQUIREMENTS更新）      │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 2: 計画策定 (🤖 AUTO)                                  │
│   ブランチ作成 → コードベース調査 → 影響分析 →              │
│   実装計画書作成 → Draft PR作成                              │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 3: 実装 (🤖 AUTO)                                      │
│   計画に基づきコード実装 + ドキュメント同時更新               │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 4: 自動品質チェック (🤖 AUTO)                           │
│   lint & テスト & 静的解析（並列実行）                       │
│   ↓ FAIL → 自動修正 → Step 4 再実行（最大3回）             │
│   ↓ 3回FAIL → Step 7へ（人間にエスカレーション）            │
└─────────────────────────────────────────────────────────────┘
    ↓ PASS
┌─────────────────────────────────────────────────────────────┐
│ Step 5: AIクロスレビュー (🤖 AUTO)                           │
│   3ペルソナ以上による多角的レビュー（YAML構造化出力）         │
│   ↓ Critical → 自動修正 → Step 4 から再実行                │
│   ↓ Minor only → 自動修正 → Step 5 再実行                  │
└─────────────────────────────────────────────────────────────┘
    ↓ PASS
┌─────────────────────────────────────────────────────────────┐
│ Step 6: PR仕上げ (🤖 AUTO)                                  │
│   コミット整理 → プッシュ → PR説明文更新 → PR Ready化        │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 7: 最終承認 (🔴 HUMAN GATE)                            │
│   人間（殿）はAIレビューサマリを確認。承認 → マージ           │
│   差戻し → Step 3 に戻り（人間の指摘を反映して再実装）       │
└─────────────────────────────────────────────────────────────┘
```

## `/workflow` コマンド

| コマンド | 説明 |
|---------|------|
| `/workflow start <説明>` | 新規ワークフロー開始（Step 1: 要件確認から） |
| `/workflow status` | 現在の状態・進捗確認 |
| `/workflow next` | 次のステップを自動実行 |
| `/workflow auto` | Step 2〜6を自動連続実行（要件確定後に使用） |
| `/workflow resume <id>` | 中断から再開 |
| `/workflow approve` | Step 7: 最終承認（マージ実行） |
| `/workflow reject <理由>` | Step 7: 差戻し（Step 3に戻る） |
| `/workflow report` | 全レポートのサマリー表示 |
| `/workflow skip-code` | ドキュメントのみ変更時にコード実装をスキップ |

## 小規模修正モード

条件: 変更ファイル数 ≤ 3 かつ 変更行数 ≤ 50

```text
Step 1（簡易） → Step 3 → Step 4 → Step 5 → Step 6 → Step 7
（Step 2: 計画策定を省略）
```
